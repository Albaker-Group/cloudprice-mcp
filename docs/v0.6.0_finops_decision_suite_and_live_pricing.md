# v0.6.0 — FinOps Decision Suite + Live Pricing (in flight)

**Status:** Active build, started 2026-05-09. Single bundled release. **One ship, one announcement.**
**Estimated effort:** ~65 hours over ~7 weeks of evenings.
**Author:** Ali Albaker.

---

## Why this scope

Two parallel needs validated by real usage:

1. **FinOps decision tools** — the morning queries on 2026-05-09 (RI break-even on 6× t3.2xlarge, cold-storage migration payback) prototyped the value end-to-end via Copilot. Naming the analyses as named tools removes the AI-chaining friction and makes them callable from CLI / scripts as well.
2. **Live pricing mode** — kills the #1 critique of catalog-snapshot tools forever ("your data is stale"). Makes the FinOps tools more credible. OCI's public pricing API already verified working during v0.5.0 development.

Bundling both into v0.6.0 means one announcement, one major-leap story instead of two incremental ones.

---

## Goals

By the end of v0.6.0 a user can:

- Run a single MCP tool call to ask *"Should I move this workload? When does an RI pay back? What's my 3-year TCO across clouds?"* and get a structured, paste-ready answer.
- Run the same analyses from a CLI without an AI client (`cloudprice-mcp finops assess workloads.yaml`).
- Optionally enable live pricing (`--live` flag) to fetch real-time SKU rates from each cloud's public API.
- Pin the analysis to a specific point-in-time catalog for reproducibility (`--catalog`).

---

## The four FinOps decision tools

### 1. `assess_migration`
The "should I move?" tool. Inputs: source cloud + workload inventory + target clouds. Outputs: per-target monthly cost, savings %, exit cost, payback months, ranked recommendation, caveats hits.

### 2. `optimize_commitment`
The "when does my RI pay back?" tool. Inputs: workload inventory + commitment scenarios. Outputs: per-scenario monthly cost, monthly savings, upfront cost, payback months, 3-year TCO, recommendation. Uses **per-family RI tiers** (3 families × 4 clouds = 12 representative rates) instead of v0.5's flat 30%/50%.

### 3. `compare_total_cost_of_ownership`
The "what's my 3-year cost?" tool. Inputs: workload inventory + horizon years + growth assumptions. Outputs: per-cloud per-year cost projection, cumulative TCO, sensitivity analysis on key variables.

### 4. `find_egress_arbitrage`
The "where do I save on data transfer?" tool — specialized version of `assess_migration` focused on egress patterns. Inputs: source cloud + egress profile (mix of internet vs inter-region). Outputs: per-target cost, payback months, recommendation.

## Live pricing mode (`--live` flag)

```bash
cloudprice-mcp setup --live              # configure clients to use live pricing
cloudprice-mcp serve --catalog           # explicit fallback to bundled catalog
CLOUDPRICE_LIVE=1                        # env-var alternative
CLOUDPRICE_CACHE_TTL=3600                # disk-cache TTL (default 1 hour)
CLOUDPRICE_GCP_API_KEY=AIza...           # required only when querying GCP live
```

Per cloud:

| Cloud | Source | Auth | Latency |
|---|---|---|---|
| AWS | [Price List Bulk API](https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/price-changes.html) | None | ~1–2s/region/service |
| Azure | [Retail Prices API](https://learn.microsoft.com/en-us/rest/api/cost-management/retail-prices/azure-retail-prices) | None | ~500ms |
| GCP | [Cloud Billing Catalog API](https://cloud.google.com/billing/docs/reference/rest/v1/services.skus) | API key (free tier) | ~500ms |
| OCI | [Public pricing API](https://apexapps.oracle.com/pls/apex/cetools/api/v1/products/) | None | ~300ms (verified) |

**Implementation:**
- Strategy pattern in `pricing.py` — `CatalogPricingSource` (current) and `LivePricingSource` (new)
- Disk cache at `~/.cache/cloudprice-mcp/live/` with TTL-based invalidation
- Graceful fallback to bundled catalog on rate-limit / network error / API outage (logged, not silent)
- Each adapter responsible for normalizing its cloud's response into our internal SKU schema

## Decisions locked (made on 2026-05-09)

| Decision | Value | Rationale |
|---|---|---|
| Ship label | **v0.6.0** | Clean v0.5.3 → v0.6.0 sequence. No version skipping. |
| Internal build order | FinOps tools first, then live pricing layered on | Less rework. v0.6 tools work against catalog, then v0.7 swaps in live source transparently. |
| Live mode default | OFF in v0.6.0, opt-in via `--live`. Flip ON in v0.6.1 after 2-week soak. | Safe rollout. |
| GCP API key | User-provided via `CLOUDPRICE_GCP_API_KEY` env var + `cloudprice-mcp setup --gcp-live-key` helper | No silent security risk. |
| Inventory format priority | YAML first; CSV / JSON later if requested | YAML is the easiest to write by hand; saves ~3 hrs upfront. |
| RI tier research | 3 representative families per cloud (compute / general / memory) | Avoid false precision. Rates change quarterly; over-engineering is wasted work. |
| Announcement strategy | **One** announcement when v0.6.0 ships. No teaser posts. | Bigger story, more impact. Tuesday's LinkedIn is about v0.5.3 only. |
